## Features Implemented

### Bronze — Foundation
- [x] **Gmail IMAP watcher** (`gmail_watcher.py`) — polls every 2 min, converts unread emails to `.md` task files in `Needs_Action/`
- [x] **Filesystem watcher** (`filesystem_watcher.py`) — watches `Drop_Here/` using WSL-safe `PollingObserver`; any dropped file becomes a structured task
- [x] **Vault folder structure** — `Needs_Action/`, `In_Progress/`, `Pending_Approval/`, `Approved/`, `Done/`, `Plans/`, `Briefings/`, `Logs/`
- [x] **Company Handbook** (`Company_Handbook.md`) — human-readable AI rulebook; $500 financial gate, approval workflows, communication policy
- [x] **Dashboard** (`Dashboard.md`) — live activity log updated after every agent run
- [x] **`/process-files` skill** — reads every task in `Needs_Action/`, creates a `Plan.md`, then auto-executes or escalates to `Pending_Approval/`
- [x] **`/create-plan` skill** — structured YAML-frontmatter plan generator used before any action
- [x] **`email-mcp` MCP server** — Node.js server giving Claude a `send_email` tool (nodemailer/Gmail); no SMTP credentials exposed
- [x] **`run_claude.sh` cron wrapper** — non-interactive agent runner; logs to `Logs/cron.log` with timestamps

### Silver — Social Media & Automation Loop
- [x] **LinkedIn Playwright poster** (`linkedin_poster.py`) — posts text, persists session in `linkedin_session/session.json`, watch mode via `Drop_Here/linkedin_*.txt`
- [x] **Facebook + Instagram poster** (`social_poster.py`) — Chromium headless with Chrome user-agent; Facebook two-step composer flow; Instagram image+caption pairs; pre/post screenshots saved to `Logs/`
- [x] **Ralph Wiggum stop hook** (`.claude/hooks/stop_hook.py`) — self-looping agent: blocks Claude exit if `In_Progress/` is non-empty, re-injects `/process-files`, safety ceiling of 10 iterations; logs every decision to `Logs/ralph_wiggum.log`
- [x] **`In_Progress/` mid-flight tracking** — tasks moved here are guaranteed to be re-processed before Claude exits
- [x] **`/ceo-briefing` skill** — full Monday morning CEO report: exec summary, weekly stats table, completed items, pending actions, bottlenecks, proactive suggestions, system health; archived to `Briefings/YYYY-MM-DD_CEO_Briefing.md`

### Platinum — Cloud VM 24/7 Deployment
- [x] **Cloud VM deployment** — AI Employee runs on an always-on Oracle Cloud VM (Ubuntu), not a developer laptop
- [x] **PM2 process management** (`scripts/setup_pm2.sh`) — `gmail_watcher` and `filesystem_watcher` managed by PM2; auto-restart on crash, startup persistence across reboots
- [x] **Vault sync** (`scripts/vault_sync.sh`) — rsync-based bidirectional vault synchronization for cloud ↔ local backup; cron-schedulable
- [x] **Health monitoring** (`scripts/health_monitor.sh`) — checks PM2 process status, disk usage, recent log activity, and sends alert emails when services are degraded
- [x] **Cloud setup automation** (`scripts/cloud_setup.sh`) — one-shot provisioning script for fresh Ubuntu VM: installs uv, Node.js, PM2, Playwright deps, configures env vars
- [x] **Env-based VAULT_PATH** — all scripts and hooks use `$VAULT_PATH` env var instead of hardcoded WSL paths; works identically on cloud VM and local dev
- [x] **Stop hook cloud-ready** — Ralph Wiggum hook updated to use `VAULT_PATH` and correct cloud paths, verified running on live VM

### Gold — CRM Integration & Twitter
- [x] **Twitter/X Playwright poster** (`twitter_poster.py`) — Firefox headless (harder to fingerprint than Chromium); handles Twitter's 3-step login flow (email → username verify → password); step screenshots to `Logs/twitter_*.png`; 280-char guard; watch mode via `Drop_Here/twitter_*.txt`
- [x] **`odoo-mcp` MCP server** — Node.js MCP server exposing Odoo CRM/ERP as Claude tools: `create_lead`, `get_leads`, `update_lead`, `create_contact`, `get_contacts`; Claude can interact with a live Odoo instance by natural language
- [x] **`Business_Goals.md`** — KPI targets file (revenue goal, lead targets, social following goals) consumed by `/ceo-briefing` to report progress vs. plan
- [x] **`Briefings/` archive** — dated CEO briefing reports generated by `/ceo-briefing`
- [x] **`/ceo-audit` skill** — structured audit of all vault folders and agent health for compliance review

## Demo Instructions

### Prerequisites

- Python 3.11+ with [uv](https://docs.astral.sh/uv/)
- Node.js 18+
- Claude Code CLI
- WSL2 (Ubuntu) recommended

### 1. Install dependencies

```bash
cd ai-employee
uv sync
uv run playwright install chromium
uv run playwright install firefox

cd ../mcp-servers/email-mcp && npm install
cd ../odoo-mcp && npm install
```

### 2. Set environment variables

```bash
export GMAIL_APP_PASSWORD="your-16-char-app-password"
export LINKEDIN_EMAIL="you@email.com"
export LINKEDIN_PASSWORD="your-linkedin-pass"
export FACEBOOK_EMAIL="you@email.com"
export FACEBOOK_PASSWORD="your-fb-pass"
export INSTAGRAM_EMAIL="you@email.com"
export INSTAGRAM_PASSWORD="your-ig-pass"
export TWITTER_EMAIL="you@email.com"
export TWITTER_PASSWORD="your-twitter-pass"
export TWITTER_USERNAME="yourhandle"
# Optional — Gold CRM tier
export ODOO_URL="https://your-odoo.com"
export ODOO_DB="your-db"
export ODOO_USER="admin"
export ODOO_PASSWORD="your-odoo-pass"
```

### 3. Quickstart: Task Processing Loop

```bash
# Drop an email-like task file
echo "Please order 10 boxes of pens — approx $45 total" > /mnt/c/AI_Employee_Vault/Drop_Here/purchase_request.txt

# Run the agent (processes Needs_Action/, updates Dashboard.md)
claude "/process-files"
```

Watch Claude: read the task → check the Handbook → create a Plan → auto-approve (under $500) → move to `Done/` — then Ralph Wiggum fires and confirms nothing is left in `In_Progress/`.

### 4. Social Media: Post to LinkedIn

```bash
cd ai-employee
uv run python linkedin_poster.py "Excited to share our latest AI project update! #AI #automation"
```

Or watch mode — drop a file and walk away:
```bash
echo "Watch mode post content" > /mnt/c/AI_Employee_Vault/Drop_Here/linkedin_demo.txt
uv run python linkedin_poster.py --watch
```

### 5. Social Media: Post to Facebook / Instagram

```bash
# Facebook
uv run python social_poster.py --platform facebook --content "Hello from the AI Employee!"

# Instagram (image required)
uv run python social_poster.py --platform instagram --content "Caption here" --image /path/to/photo.jpg
```

### 6. CEO Briefing

```bash
claude "/ceo-briefing"
# Report written to Briefings/2026-02-28_CEO_Briefing.md
```

### 7. Odoo CRM (Gold Tier)

Once `odoo-mcp` is configured in `.claude/settings.json`:
```
You: "Create a CRM lead for Acme Corp, contact John Smith, potential value $12,000"
Claude: [calls create_lead tool] → Lead created in Odoo
```

## Known Issues

### Twitter/X — IP Rate Limit

**Status:** Script implemented and complete. Blocked by Twitter IP-level rate limiting in the current network environment.

**Symptoms:** Login attempt returns "Could not log you in now" error immediately after entering credentials. Step-debug screenshots saved to `Logs/twitter_*.png` confirm the flow reaches Twitter's server-side block, not a code error.

**Root cause:** Twitter aggressively rate-limits automated login attempts at the IP level, particularly from cloud/VPS/WSL IPs. This is a platform restriction, not a code defect.

**Resolution path:**
1. Run from a residential IP or use a mobile proxy
2. Wait 60+ minutes between attempts (rate limit resets)
3. Pre-authenticate once via `--debug` on a clean IP to save `sessions/twitter_session.json`; all subsequent runs skip login entirely

**The code is correct:** Firefox headless, `navigator.webdriver` masked, 3-step login flow (email → username verify → password), 280-char guard, watch mode, session persistence — all implemented and verified against the flow up to the IP block point.

## What Makes This Project Stand Out

### 1. A genuine autonomous agent loop — not a demo script

Most hackathon AI projects run a single prompt and return an answer. This project implements a **persistent, self-healing agent loop**: Ralph Wiggum's stop hook means Claude literally cannot exit until its work is done. Drop a task file and go to lunch — it will be processed.

### 2. Rulebook-governed decision making

The `Company_Handbook.md` is not a system prompt — it's a **living document** that Claude reads at runtime. Change the $500 approval threshold to $100, or add a new rule, and Claude's behavior changes immediately without any code changes.

### 3. Four social platforms from a single drop folder

Dump `linkedin_hello.txt`, `facebook_promo.txt`, `instagram_launch.txt + instagram_launch.jpg`, and `twitter_news.txt` into `Drop_Here/` and walk away. Four platforms, one folder, zero additional configuration per post.

### 4. MCP servers as the integration layer

Rather than hard-coding Gmail or Odoo API calls into the agent prompts, this project exposes them as **MCP tool calls**. Claude uses `send_email` and `create_lead` as naturally as it uses any other tool — and the MCP servers handle auth, retries, and error formatting out of view.

### 5. CEO-facing reporting out of the box

The `/ceo-briefing` skill turns the vault's folder state into a **structured executive report** — stats table, overdue items flagged, bottlenecks identified, proactive suggestions with file references — in one command. Most automation projects have no reporting story at all.

### 6. Zero hardcoded secrets, no `.env` files committed

Every credential is a runtime environment variable. Browser sessions (which contain auth cookies) are gitignored. Gmail OAuth tokens are gitignored. The codebase is safe to open-source as-is.

### 7. WSL-first engineering

Every sharp edge of running Playwright and Watchdog under WSL2 has been solved: `PollingObserver` for reliable file events, `--no-sandbox --disable-dev-shm-usage` for Chromium, Firefox headless for Twitter's stricter bot detection.

### 8. Production cloud deployment with PM2 and health monitoring

The AI Employee runs 24/7 on a cloud VM — not just a developer laptop. PM2 ensures watchers restart on crash and survive reboots. `health_monitor.sh` checks process liveness, disk pressure, and log freshness, sending alert emails when anything looks wrong. `vault_sync.sh` provides rsync-based backup. A single `cloud_setup.sh` provisions a fresh Ubuntu VM from zero to running in minutes.
